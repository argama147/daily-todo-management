description: 仕様Issueをテンプレートに沿って作成する
---

以下の手順で仕様Issueを作成してください。

## 手順

### 1. 内容をヒアリングする

AskUserQuestion ツールを使って以下を一括で確認する:

- **タイトル**: Issue のタイトル（`[仕様]` プレフィックスは自動付与）
- **概要**: どのような仕様を追加・変更するか、背景や目的
- **追加・変更内容**: 箇条書きで
- **スコープ外**: 変更しない範囲（不要なら「なし」）
- **備考**: 実装時の考慮事項（不要なら「なし」）

### 2. 内容を提示して確認を取る

以下の形式で表示し、AskUserQuestion で確認する:

```
---
## 作成予定の仕様Issue

**タイトル:** [仕様] <title>

**本文:**
## 概要
<overview>

## 追加・変更内容
<changes>

## 変更しない範囲（スコープ外）
<out_of_scope>

## 備考
<notes>
---
この内容でIssueを作成しますか？
```

### 3. 同意が得られた場合のみIssueを作成する

```bash
gh issue create \
  --title "[仕様] <title>" \
  --label spec \
  --assignee argama147 \
  --body "$(cat <<'EOF'
## 概要
<overview>

## 追加・変更内容
<changes>

## 変更しない範囲（スコープ外）
<out_of_scope>

## 備考
<notes>
EOF
)"
```

- 作成後はIssueのURLをユーザーに表示する
- 却下された場合は修正希望を聞く
